# Hot Monitor — 需求文档

> **版本**: v1.3 | **日期**: 2026-06-25

---

## 1. 项目背景

AI 编程博主需要第一时间获取热点动态（如 AI 大模型更新），不能依赖人工手动搜索。需要一个工具自动发现指定热点变化，及时通知。

定位：轻量级工具类项目，敏捷开发，不过度工程化。

---

## 2. 用户角色

- **AI 编程博主**：添加监控关键词，查看实时热点，第一时间获取通知
- **系统**：定时自动扫描、AI 验证真伪、存储并推送

---

## 3. 功能需求

### F1: 关键词监控

| 功能 | 说明 |
|------|------|
| 添加关键词 | 输入关键词 + 可选监控范围（如 "GPT-5"、"AI 编程"） |
| 滑块开关 | 绿色滑块激活/暂停监控 |
| 编辑 | 修改关键词名称和范围 |
| 删除 | 确认后删除 |

### F2: AI 多维验证

| 功能 | 说明 |
|------|------|
| 内容验证 | 判断搜索结果是否真正与关键词相关 |
| 假冒识别 | 排除同名不同义、蹭热度、标题党 |
| 相关性评分 (R) | 0-100 分 |
| 重要性评分 (I) | 0-100 分，评估新闻价值/讨论热度 |
| 时效性评分 (F) | 0-100 分，评估内容新鲜度（热点核心指标） |
| 摘要生成 | AI 生成一句话中文摘要 |

> 综合分 = (R + I + F) / 3，低于来源阈值不入库。

### F3: 多源搜索

12 大免费数据源，无需 API Key：

| 来源 | 类型 | 说明 |
|------|------|------|
| Bing | 国际搜索 | 网页爬虫 |
| Google | 国际搜索 | 网页爬虫 |
| DuckDuckGo | 隐私搜索 | Lite HTML |
| 搜狗 | 中文搜索 | 网页爬虫 |
| 百度 | 中文搜索 | 网页爬虫 |
| HackerNews | 科技社区 | Algolia API |
| B站 | 视频平台 | 官方 API |
| 微博 | 社交媒体 | 网页爬虫 + Ajax 热搜 API |
| GitHub | 开发者社区 | 官方 API（无 Key） |
| 掘金 | 开发者社区 | 官方 API |
| 知乎 | 问答社区 | 网页爬虫 |
| Reddit | 国际社区 | .json API |

> `searchAll` 轮询取结果（round-robin），确保来源多样性。

### F4: 三层质量过滤

| 层级 | 名称 | 说明 |
|:---:|------|------|
| L1 | 互动量预过滤 | HN 赞≥10/评论≥5；B站 播放≥500；GitHub Stars≥10/Follower≥5；掘金 点赞≥5；Reddit 得分≥10/评论≥3；微博热搜 热度≥10万 |
| L2 | 时效预过滤 | 搜索源返回 `pub_date` 且 > `max_age_days`（默认 7 天）→ 直接丢弃，省 AI token |
| L3 | AI 三维验证 | `isRelevant` + `isFake` + `R` + `I` + `F`；F<40 硬拒绝；综合分 ≥ 来源阈值 |

> 来源分类：搜索引擎（Bing/Google/DDG/搜狗/百度/知乎/微博搜索）阈值 70；社区源（HN/B站/GitHub/掘金/Reddit/微博热搜）阈值 55。

### F5: 账号检测

| 平台 | 检测内容 |
|------|---------|
| GitHub | 搜索匹配 User/Org → 拉取 followers/public_repos/avatar |
| B站 | `detectBilibiliAccount(mid)` → followers/official/level（预留） |
| 知乎 | `detectZhihuAccount(urlToken)` → followers/headline（预留） |

### F6: 定时扫描

| 功能 | 说明 |
|------|------|
| 自动扫描 | 每 N 分钟自动扫描所有活跃关键词 |
| 手动扫描 | Header 按钮触发全量扫描，轮询等待完成（最长 2 分钟） |
| 首次扫描 | 启动后 10 秒自动执行 |

### F7: 通知

| 功能 | 说明 |
|------|------|
| 页面轮询 | 15 秒间隔刷新热点流 |
| 新热点计数 | Header 红色 badge |
| 邮件通知 | SMTP 配置后自动发送（待配置） |

### F8: Web 管理界面

| 页面/区域 | 功能 |
|-----------|------|
| 热点流（首页） | 实时展示已验证热点，环形综合评分，R:I:F 三维显示 |
| 监控词 | 关键词管理（添加/编辑/删除/开关） |
| 搜索 | 搜索历史热点（标题/摘要/关键词全文检索） |
| 统计卡片 | 关键词数、热点数、24h、AI 验证数 |
| 系统日志 | Header 按钮 → 点击弹窗查看 |
| 设置 | 扫描间隔、过滤阈值（搜索引擎/社区源）、内容最大天数、邮件 SMTP |

### F9: Agent Skills（Phase 2）

封装为 Agent Skills，供其他 AI Agent 调用。

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 部署 | 单命令启动，零外部依赖 |
| 存储 | 本地 SQLite，无需安装数据库服务 |
| 性能 | 前端响应 < 200ms，12 源搜索+AI < 3min |
| 安全 | API Key 通过 .env 管理，.gitignore 排除 |
| 可扩展 | 搜索源可插拔，通知渠道可扩展，过滤策略可配置 |
| UI | 深空玻璃风格，Aceternity 动效，响应式 |
