# Hot Monitor — 需求文档

> **版本**: v1.5 | **日期**: 2026-06-27

---

## 1. 项目背景

AI 编程博主需要第一时间获取热点动态（如 AI 大模型更新），不能依赖人工手动搜索。需要一个工具自动发现指定热点变化，及时通知。

定位：轻量级工具类项目，敏捷开发，不过度工程化。

---

## 2. 用户角色

- **AI 编程博主**：添加监控关键词，查看实时热点，第一时间获取通知
- **系统**：定时自动扫描、AI 验证真伪、存储并推送

---

## 3. 功能需求

### F1: 关键词监控

| 功能 | 说明 |
|------|------|
| 添加关键词 | 输入关键词 + 可选监控范围（如 "GPT-5"、"AI 编程"） |
| 滑块开关 | 绿色滑块激活/暂停监控 |
| 编辑 | 修改关键词名称和范围 |
| 删除 | 确认后删除 |

### F2: AI 多维验证

| 功能 | 说明 |
|------|------|
| 内容分类 | AI 先判断内容类型：article / announcement / discussion / profile / product / mention / spam |
| 商业页面识别 | 电商/商品/课程/广告页面 → 自动降低相关性评分 |
| 直接关联判断 | 区分"核心内容围绕关键词主体"和"仅顺便提及关键词" |
| 内容验证 | 判断搜索结果是否真正与关键词相关 |
| 假冒识别 | 排除同名不同义、蹭热度、标题党 |
| 相关性评分 (R) | 0-100 分，R<40 硬拒绝（独立于综合分） |
| 重要性评分 (I) | 0-100 分，评估新闻价值/讨论热度 |
| 时效性评分 (F) | 0-100 分，评估内容新鲜度（热点核心指标） |
| 判断理由 | AI 输出 20-40 字相关性判断理由，含内容类型前缀 |
| 摘要生成 | AI 生成一句话中文摘要（上限 120 字符） |

> 综合分 = (R + I + F) / 3，低于来源阈值不入库。R < 40 或 F < 40 硬拒绝。

### F2b: 查询扩展

| 功能 | 说明 |
|------|------|
| AI 查询扩展 | 将关键词扩展为 3-5 个相关搜索词（近义词/变体/换序/中英文） |
| 多词搜索 | 对扩展后的查询列表逐一搜索，URL+标题去重合并 |
| 提高覆盖率 | 避免"鱼皮的AI导航"搜不到"程序员鱼皮AI导航"的合理内容 |

### F3: 多源搜索

12 大免费数据源，无需 API Key：

| 来源 | 类型 | 说明 |
|------|------|------|
| Bing | 国际搜索 | 网页爬虫，多段摘要合并 |
| Google | 国际搜索 | 网页爬虫 |
| DuckDuckGo | 隐私搜索 | Lite HTML |
| 搜狗 | 中文搜索 | 网页爬虫，扩展选择器 |
| 百度 | 中文搜索 | 网页爬虫，多段摘要合并 |
| HackerNews | 科技社区 | Algolia API，抓取 `story_text` 帖子正文 |
| B站 | 视频平台 | 官方 API，抓取 `description` 视频描述 |
| 微博 | 社交媒体 | 网页爬虫 + Ajax 热搜 API，抓取微博正文 |
| GitHub | 开发者社区 | 官方 API（无 Key） |
| 掘金 | 开发者社区 | 官方 API，抓取 `brief_content` 文章摘要 |
| 知乎 | 问答社区 | 网页爬虫 |
| Reddit | 国际社区 | .json API，抓取 `selftext` 帖子正文 |

> `searchAll` 轮询取结果（round-robin），确保来源多样性。社区源优先抓取实际描述文本，降级使用统计数据。

### F4: 三层质量过滤 + 去重

| 层级 | 名称 | 说明 |
|:---:|------|------|
| L1 | 互动量预过滤 | HN 赞≥10/评论≥5；B站 播放≥500；GitHub Stars≥10/Follower≥5；掘金 点赞≥5；Reddit 得分≥10/评论≥3；微博热搜 热度≥10万 |
| L2 | 时效预过滤 | 搜索源返回 `pub_date` 且 > `max_age_days`（默认 7 天）→ 直接丢弃，省 AI token |
| L3 | AI 多维验证 | 内容分类 + `isRelevant` + `isFake` + `R` + `I` + `F` + `reason`；R<40 或 F<40 硬拒绝；综合分 ≥ 来源阈值 |
| — | **URL 规范化去重** | 提取 Google/Baidu/Sogou 重定向中的真实 URL，剥离追踪参数，归一化比对 |
| — | **标题归一化去重** | 小写+去标点+截断 80 字后比对，防止同一文章从不同搜索引擎多次入库 |

> 来源分类：搜索引擎（Bing/Google/DDG/搜狗/百度/知乎/微博搜索）阈值 70；社区源（HN/B站/GitHub/掘金/Reddit/微博热搜）阈值 55。

### F5: 账号检测

| 平台 | 检测内容 |
|------|---------|
| GitHub | 搜索匹配 User/Org → 拉取 followers/public_repos/avatar |
| B站 | `detectBilibiliAccount(mid)` → followers/official/level（预留） |
| 知乎 | `detectZhihuAccount(urlToken)` → followers/headline（预留） |

### F6: 定时扫描

| 功能 | 说明 |
|------|------|
| 自动扫描 | 每 N 分钟自动扫描所有活跃关键词 |
| 手动扫描 | Header 按钮触发全量扫描，轮询等待完成（最长 2 分钟） |
| 首次扫描 | 启动后 10 秒自动执行 |

### F7: 通知

| 功能 | 说明 |
|------|------|
| Toast 实时弹出 | 新热点被捕获后页面右上角弹出通知，显示标题+来源，4 秒自动消失 |
| 铃铛通知列表 | Header 铃铛图标显示未读数 badge，点击展开最近 20 条新热点列表 |
| 页面轮询 | 15 秒间隔刷新热点流，自动检测新增 ID 并触发通知 |
| 邮件通知 | SMTP 配置后自动发送（待配置） |

### F8: Web 管理界面

| 页面/区域 | 功能 |
|-----------|------|
| 热点流（首页） | 实时展示已验证热点，环形综合评分，R:I:F 三维显示，展开/折叠全部 AI 分析 |
| 监控词 | 关键词管理（添加/编辑/删除/开关） |
| 搜索 | 全文检索历史热点（标题/摘要/关键词），支持排序/筛选/分页 |
| 统计卡片 | 关键词数、热点数、24h、AI 验证数 |
| 新热点通知 | Header 铃铛图标 + 未读计数 badge → 点击展开通知列表 |
| AI 审核评估 | 设置面板内，抽样重评历史记录，输出准确率+偏差详情 |
| 设置 | 扫描间隔、过滤阈值（搜索引擎/社区源）、内容最大天数、邮件 SMTP |

### F9: Agent Skills（Phase 2）

封装为 Agent Skills，供其他 AI Agent 调用。

### F10: 热点卡片信息增强

| 功能 | 说明 |
|------|------|
| AI 摘要展示 | AI 提炼一句话摘要（超 80 字截断 + hover tooltip 显示全文） |
| 原始描述预览 | 搜索引擎结果原文描述，可折叠展开（统计源自动隐藏） |
| 互动数据展示 | 按来源展示点赞/播放/Star/评论等（HN/B站/GitHub/掘金/Reddit/知乎/微博热搜） |
| 作者展示 | 显示内容作者/UP主 |
| 发布时间展示 | 显示内容原始发布时间 + 系统抓取时间 |
| AI 理由展开 | 点击展开 AI 相关性判断理由，支持全局一键展开/折叠 |
| 原文跳转 | 标题为独立可点击链接（↗ 图标），卡片其余区域支持文本选中复制 |

### F11: 排序与筛选

| 维度 | 选项 |
|------|------|
| 排序字段 | 发现时间、综合评分、相关性 R、重要性 I、时效性 F、来源 |
| 排序方向 | 升序 / 降序 |
| 时间范围 | 1h / 24h / 7天 / 30天 / 全部 |
| 来源平台 | 12 平台多选 |
| 来源类型 | 搜索引擎 / 社区源 |
| 关键词 | 多选监控词 |
| 综合评分区间 | 快捷（≥80 / 60-79 / <60）+ 自定义范围 |
| R/I/F 评分区间 | 0-100 自定义范围 |
| AI 验证状态 | 已验证 / 未验证 / 全部 |
| 通知状态 | 已通知 / 未通知 / 全部 |
| 分页 | 服务端分页，每页 20 条，首页/末页/上下页/页码导航 |

> 筛选栏常驻显示，切换任一筛选自动重置到第 1 页。搜索页额外支持 15 条/页客户端分页。

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 部署 | 单命令启动，零外部依赖 |
| 存储 | 本地 SQLite，无需安装数据库服务 |
| 性能 | 前端响应 < 200ms，12 源搜索+AI < 3min |
| 安全 | API Key 通过 .env 管理，.gitignore 排除 |
| 可扩展 | 搜索源可插拔，通知渠道可扩展，过滤策略可配置 |
| UI | 深空玻璃风格，Aceternity 动效，响应式，页面自然滚动 |
